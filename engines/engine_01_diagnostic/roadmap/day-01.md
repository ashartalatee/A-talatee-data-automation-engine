# Day 01 — Foundation & Engine Philosophy

## Tujuan Hari Ini
Membangun fondasi Raw Data Diagnostic Engine sebagai sistem profesional,
bukan kumpulan script. Hari ini fokus pada struktur, batasan, dan arah engine
agar tidak salah tumbuh di hari-hari berikutnya.

## Yang Dikerjakan
- Inisialisasi repository Git
- Menyusun struktur folder final engine
- Menentukan batas tanggung jawab engine (diagnostic ≠ cleaning)
- Menetapkan prinsip kerja: repeatable, explainable, auditable

## Output
- Folder structure engine siap dikembangkan
- File roadmap/day-01.md sebagai jejak berpikir
- Repo siap di-push ke GitHub

## Pelajaran Penting
Data bermasalah bukan karena “kotor”, tapi karena:
- tidak terdefinisi
- tidak diaudit
- tidak dipahami risikonya

Engine ini tidak memperbaiki data.
Engine ini menjelaskan kondisi data sebelum keputusan dibuat.

## Dampak ke Agency
Dengan struktur ini:
- Agency punya standar kerja (bukan improvisasi)
- Klien bisa diaudit tanpa langsung menyentuh data
- Scope kerja jelas → mudah bilang “tidak” pada data berisiko

Fondasi hari ini mencegah kekacauan di hari ke-20.
